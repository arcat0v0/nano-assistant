# nano-assistant v0.3.1

## Highlights

- Tightened runtime self-management guidance so the assistant executes operational work before narrating, records durable facts to memory, and reports outcomes more concisely
- Expanded startup system context with richer hardware and tool detection, including binary paths where available
- Fixed Unix config path handling so `~/.config/nano-assistant` resolves to the real user home instead of a literal `~/` directory

## User-facing changes

- Memory and config paths now behave correctly even when `HOME` is unavailable in the immediate process environment
- Repositories using nano-assistant can ignore common local artifacts such as `.sisyphus/`, `AGENTS.md`, `nvim.log`, and accidental `~/` scratch trees
- Full test runs are stable after isolating and serializing `HOME`-sensitive tests

## Verification

- `cargo fmt --check`
- `cargo test`
- `cargo build --release`
